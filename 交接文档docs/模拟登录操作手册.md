# 插件模拟登录调试操作手册

本手册指导如何使用本地静态页面模拟 OAuth 登录流程，用于开发调试。

## 1. 环境准备

1. **安装 Live Server 插件**：在 VS Code 扩展商店搜索并安装 `Live Server`。
2. **打开项目根目录**：确保当前 VS Code 窗口打开的是 `存放路径\continue`。

## 2. 启动模拟登录服务器

1. 在文件资源管理器中找到：`extensions/vscode/authmanager/continue_login.html`。
2. **右键点击**该文件，选择 **"Open with Live Server"**。
3. 浏览器会自动打开该页面，地址通常为：`http://127.0.0.1:5500/extensions/vscode/authmanager/continue_login.html`。
   *注：如果端口不是 5500，请同步修改 config.yaml 中的 appUrl。*

## 3. 配置文件检查

确认 `c:\Users\用户名\.cowork\config.yaml` 中的配置如下：

```yaml
controlPlane:
  appUrl: "http://127.0.0.1:5500/extensions/vscode/authmanager/continue_login.html"
```

## 4. 执行登录流程

1. **启动插件调试**：按 `F5` 启动 VS Code 扩展开发宿主窗口。
2. **触发登录**：
   - 在侧边栏打开 Continue 插件。
   - 点击界面上的 **"登录"** 按钮。
3. **完成授权**：
   - 插件会自动在浏览器中打开模拟登录页。
   - 页面会自动解析 `state` 和 `callback` 参数。
   - 点击页面上的 **"确认登录"** 按钮。
4. **验证结果**：
   - 浏览器会重定向回插件本地服务（34567 端口）。
   - VS Code 右下角应弹出 “登录成功” 提示。
   - 插件侧边栏应自动跳转至聊天主界面。

## 5. 常见问题

- **端口冲突**：如果 5500 端口被占用，Live Server 会自动切换端口，请务必保持 config.yaml 与浏览器实际地址一致。
- **参数缺失**：如果直接格在浏览器打开该页面（不通过插件点击），会提示“缺少必要参数”，这是正常现象。
