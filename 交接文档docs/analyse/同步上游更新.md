# 同步上游仓库 (luoweb/continue) 更新指南

本文档介绍如何将你 Fork 的仓库与原作者 `luoweb/continue` 的主仓库保持同步。

## 1. 配置上游仓库 (仅需操作一次)

在本地终端中运行以下命令，将原作者仓库添加为 `upstream`（上游）：

```powershell
# 添加上游仓库链接
git remote add upstream https://github.com/luoweb/continue.git

# 验证远程仓库列表，你应该能看到 origin (你的) 和 upstream (原作者的)
git remote -v
```

## 2. 获取并同步更新

当上游仓库有新代码发布时，执行以下步骤进行同步：

```powershell
# 1. 获取上游仓库的所有更新
git fetch upstream

# 2. 确保你在主分支 (通常是 main)
git checkout main

# 3. 将上游的更新合并到本地主分支
git merge upstream/main

# 4. 将更新后的代码推送到你自己的 GitHub Fork 仓库 (origin)
git push origin main
```

## 3. 处理冲突 (如有)

如果在同步过程中出现代码冲突：

1. Git 会提示哪些文件存在冲突。
2. 打开冲突文件，手动编辑并解决冲突。
3. 标记已解决的文件：`git add <文件名>`。
4. 完成合并提交：`git commit -m "merge: 同步上游更新并解决冲突"`。

## 4. 快捷方式 (GitHub 网页端)

你也可以直接通过 GitHub 网页完成同步：

1. 打开你的 GitHub Fork 仓库页面。
2. 点击 **Sync fork** 下拉菜单。
3. 点击 **Update branch**。
4. 同步完成后，在本地执行 `git pull origin main` 即可获取最新代码。
