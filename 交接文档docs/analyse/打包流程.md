# Continue VS Code 插件打包流程指南

本文档详细记录了将 Continue 项目打包为 VS Code 插件（.vsix 文件）的完整流程、遇到的问题及其解决方案。

## 1. 基础准备

在开始打包之前，请确保已安装以下环境：
- **Node.js**: 建议版本 v20+
- **npm**: 随 Node.js 安装
- **Git**: 用于版本控制和依赖管理

## 2. 标准打包步骤

### 第一步：安装全局与模块依赖
由于项目是 Monorepo 结构，需要安装根目录及核心子模块的依赖。
```powershell
# 安装根目录依赖
npm install

# 安装核心模块依赖（跳过原生模块编译以避免报错）
cd core; npm install --ignore-scripts
cd ../gui; npm install
cd ../extensions/vscode; npm install
```

### 第二步：构建 GUI (React 前端)
Continue 的聊天界面和配置界面是在 `gui` 目录下开发的，需要先编译成静态资源。
```powershell
cd gui
npm run build
```
编译结果将存放在 `gui/dist` 目录。

### 第三步：预打包准备 (Prepackage)
运行 `extensions/vscode` 目录下的 `prepackage.js` 脚本。该脚本负责将 GUI 资源、二进制文件（如 SQLite, Ripgrep, LanceDB）以及 WASM 模块拷贝到插件输出目录。
```powershell
cd extensions/vscode
npm run prepackage
```

### 第四步：代码捆绑 (Bundling)
使用 `esbuild` 将插件的后端 TypeScript 代码打包成单文件。
```powershell
cd extensions/vscode
npm run esbuild
```
打包结果为 `extensions/vscode/out/extension.js`。

### 第五步：生成 VSIX 文件
最后使用 `vsce` 工具将所有资源封装成插件安装包。
```powershell
cd extensions/vscode
npm run package
```
生成的插件位于 `extensions/vscode/build/continue-x.x.x.vsix`。

## 3. 遇到的问题及解决方案

### 问题 A：原生模块 (SQLite3) 编译失败
**现象**：在 Windows 环境下，`npm install` 尝试编译 `sqlite3` 时因缺少 C++ 编译环境而报错。
**解决**：
1. 使用 `--ignore-scripts` 跳过自动编译。
2. 手动从 GitHub 下载对应平台的预编译二进制文件。
3. 将下载的 `node_sqlite3.node` 放置到 `core/node_modules/sqlite3/build/Release/` 目录下。

### 问题 B：缺失 Ripgrep 二进制文件
**现象**：`prepackage` 脚本报错，提示 `node_modules/@vscode/ripgrep/bin/rg.exe` 不存在。
**解决**：
手动触发 `ripgrep` 的安装脚本：
```powershell
cd extensions/vscode/node_modules/@vscode/ripgrep
node ./lib/postinstall.js
```

### 问题 C：网络连接超时 (GitHub 下载失败)
**现象**：在运行 `prepackage` 时，由于网络原因无法从 GitHub 下载 SQLite 或 esbuild 的预编译包。
**解决**：
1. 检查代理设置。
2. 若代理无效，可手动通过浏览器下载文件，并解压到脚本指定的目录（例如 `core/node_modules/sqlite3/build`）。
3. 使用 `$env:SKIP_INSTALLS="true"` 环境变量运行打包脚本，跳过自动下载步骤。

### 问题 D：esbuild 缺少 meta.json 目录
**现象**：`npm run esbuild` 报错提示无法写入 `./build/meta.json`，因为 `build` 文件夹不存在。
**解决**：
手动创建 `build` 文件夹或先运行 `npm run package`（该脚本会自动创建目录）。

## 4. 安装插件

1. 打开 VS Code。
2. 进入插件视图 (`Ctrl+Shift+X`)。
3. 点击右上角 `...` 菜单 -> **Install from VSIX...**。
4. 选择生成的 `.vsix` 文件完成安装。
